# Группировка маршрутов в Expo Router

### Группировка маршрутов

Папки в скобках (например, `(tabs)`, `(auth)`) используются для организации маршрутов без влияния на URL-путь. Это позволяет логически группировать связанные маршруты и определять для них общие настройки навигации.

#### Примеры группировок:

```
app/
  (tabs)/           # Группа вкладок, не влияет на URL
    _layout.tsx     # Настройки TabNavigator для всех вкладок
    index.tsx       # URL: /
    profile.tsx     # URL: /profile

  (auth)/           # Группа авторизации
    _layout.tsx     # Настройки StackNavigator для авторизации
    login.tsx       # URL: /login
    register.tsx    # URL: /register

  (admin)/          # Группа админ-панели
    _layout.tsx     # Отдельные настройки навигации
    dashboard.tsx   # URL: /dashboard
    users.tsx       # URL: /users
```

#### Преимущества группировки:

1. **Организация кода**:

   - Логическое разделение маршрутов по функциональности
   - Улучшение навигации по проекту
   - Упрощение поддержки кода

2. **Изоляция навигации**:

   - Каждая группа может иметь свой тип навигации (`Stack`, `Tabs`, `Drawer`)
   - Отдельные настройки для каждой группы через `_layout.tsx`
   - Независимая конфигурация заголовков, переходов и анимации

3. **Чистые URL**:
   - Скобки не отображаются в итоговом URL
   - URL остаются простыми и понятными
   - Пример: файл `(auth)/login.tsx` доступен по пути `/login`

#### Вложенные группы:

```
app/
  (tabs)/
    _layout.tsx
    (feed)/           # Вложенная группа внутри tabs
      _layout.tsx     # Настройки для feed
      popular.tsx     # URL: /popular
      recent.tsx      # URL: /recent
    (profile)/        # Ещё одна вложенная группа
      _layout.tsx
      settings.tsx    # URL: /settings
```

Вложенные группы позволяют создавать сложные структуры навигации, где каждая группа может иметь свои настройки и маршруты. Это особенно полезно для крупных приложений с множеством экранов и функциональностей.
