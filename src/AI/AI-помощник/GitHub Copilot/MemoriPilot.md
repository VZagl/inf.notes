## Обзор

MemoriPilot решает проблему потери контекста между сессиями работы с GitHub Copilot. Оно создаёт и поддерживает структурированную базу знаний о проекте, которая:

- Сохраняется между сессиями.
- Фиксирует важные решения и их обоснования.
- Отслеживает прогресс и приоритеты.
- Документирует архитектуру системы и шаблоны.
- Автоматически обновляется по мере развития проекта.

---

## Особенности

- **Интеграция с GitHub Copilot**: инструменты появляются прямо в режиме агента Copilot.
- **Управление ресурсами**: предотвращение утечек памяти и безопасное удаление ресурсов.
- **Специализированные инструменты**: обновление памяти по конкретным разделам.
- **Инъекция зависимостей**: стабильность и тестируемость основных сервисов.
- **Версионирование шаблонов**: автоматическое обновление шаблонов.
- **Режимы работы**: Architect, Code, Ask, Debug — каждый с уникальным поведением.
- **Обновления в реальном времени**: поддержание согласованности между файлами.

---

## Инструменты

MemoriPilot предоставляет набор инструментов для управления памятью проекта:

- `memory_bank_update_context`: установка текущего фокуса работы.
- `memory_bank_log_decision`: запись архитектурных решений.
- `memory_bank_update_progress`: отслеживание задач (выполнено/в процессе/следующее).
- `memory_bank_show_memory`: отображение содержимого памяти.
- `memory_bank_switch_mode`: переключение режимов работы.
- `memory_bank_update_product_context`: обновление деталей продукта.
- `memory_bank_update_system_patterns`: документирование шаблонов проектирования.
- `memory_bank_update_project_brief`: уточнение требований проекта.
- `memory_bank_update_architect`: фиксация архитектурных решений.

## Установка

1. Установить из [VS Code Marketplace](https://marketplace.visualstudio.com/items?itemName=gujjar19.memoripilot).
2. Открыть папку проекта.
3. Расширение автоматически инициализируется при использовании GitHub Copilot Chat.
4. Файлы памяти создаются в директории `memory-bank/`.

## Использование

- **Естественное взаимодействие**: MemoriPilot предлагает инструменты на основе ваших запросов в Copilot Chat.
- **Прямые ссылки на инструменты**: можно использовать инструменты напрямую в запросах, например:
  ```plaintext
  memory_bank_update_context context:"Implementing user authentication flow"
  memory_bank_log_decision decision:"Use React Query" rationale:"Better caching and state management"
  ```

---

## Режимы работы

MemoriPilot поддерживает 4 режима:

1. **Architect**: проектирование системы и принятие решений.
2. **Code**: детали реализации, написание и тестирование кода.
3. **Ask**: получение знаний о проекте.
4. **Debug**: устранение проблем и отладка.

---

## Файлы памяти

MemoriPilot создаёт и поддерживает следующие файлы:

- `activeContext.md`: текущие цели и блокеры.
- `productContext.md`: обзор проекта.
- `progress.md`: отслеживание задач.
- `decisionLog.md`: архитектурные решения с временными метками.
- `projectBrief.md`: требования проекта.
- `systemPatterns.md`: шаблоны проектирования.

## Безопасность и работа MemoriPilot

MemoriPilot работает как расширение для GitHub Copilot, предоставляя инструменты для управления контекстом проекта. Вот основные аспекты его работы:

### Основные принципы работы:

1. **Создание базы знаний**:

   - MemoriPilot автоматически создаёт файлы памяти (`memory-bank/`) в проекте, такие как `activeContext.md`, `progress.md`, `decisionLog.md` и другие.
   - Эти файлы содержат информацию о текущих задачах, архитектурных решениях, шаблонах проектирования и других аспектах проекта.

2. **Интеграция с GitHub Copilot**:

   - Расширение взаимодействует с Copilot Chat, предлагая использовать инструменты MemoriPilot на основе ваших запросов.
   - Например, при упоминании задач или решений, MemoriPilot может предложить обновить соответствующий файл памяти.

3. **Режимы работы**:

   - MemoriPilot поддерживает 4 режима:
     - **Architect**: для проектирования системы и принятия архитектурных решений.
     - **Code**: для написания и тестирования кода.
     - **Ask**: для получения информации о проекте.
     - **Debug**: для устранения проблем и отладки.

4. **Обновление контекста**:

   - MemoriPilot отслеживает изменения в проекте и обновляет файлы памяти, чтобы сохранить актуальный контекст.
   - Например, при завершении задачи можно обновить файл `progress.md`, добавив её в список выполненных.

5. **Прямое использование инструментов**:
   - Вы можете напрямую использовать инструменты MemoriPilot через Copilot Chat, например:
     ```plaintext
     memory_bank_update_context context:"Implementing user authentication flow"
     memory_bank_log_decision decision:"Use React Query" rationale:"Better caching and state management"
     ```

### Пример сценария:

1. Вы работаете над проектом и решили использовать PostgreSQL для базы данных.
2. MemoriPilot предложит записать это решение в файл `decisionLog.md` с указанием причины выбора.
3. Вы завершили разработку страницы входа и начали работу над панелью управления.
4. MemoriPilot обновит файл `progress.md`, добавив выполненные и текущие задачи.

### Итог:

MemoriPilot помогает сохранять контекст проекта, фиксировать важные решения и отслеживать прогресс, что делает работу с GitHub Copilot более эффективной и организованной.
