## Получение системы

### Вариант 1: Клонирование репозитория

```bash
git clone https://github.com/vanzan01/cursor-memory-bank.git
```

## Добавление в новый проект

### Шаг 1: Подготовка проекта

1. Создайте новый проект или откройте пустую директорию
2. Убедитесь, что у вас установлен Cursor 2.0 или более новая версия

### Шаг 2: Копирование правил

**Важно:** Копируйте только необходимые файлы для работы системы. Не копируйте документацию, изображения и другие вспомогательные файлы.

**Необходимые файлы и директории:**

1. **Директория `.cursor/`** — содержит команды и правила:

   - `.cursor/commands/` — файлы команд (обязательно):
     - `van.md` — команда инициализации
     - `plan.md` — команда планирования
     - `creative.md` — команда творческого этапа
     - `build.md` — команда реализации
     - `reflect.md` — команда рефлексии
     - `archive.md` — команда архивации
   - `.cursor/rules/` — директория с правилами (копировать целиком):
     - `isolation_rules/` — вся структура правил для работы системы

2. **Файл `.cursorindexingignore`** — файл для исключения директорий из индексации Cursor

**Что НЕ нужно копировать:**

- `README.md`, `COMMANDS_README.md`, `COMMANDS_MIGRATION.md` и другая документация
- `assets/` — директория с изображениями для документации
- `optimization-journey/` — документация об оптимизации системы
- `.gitignore` — файл конфигурации Git
- `RELEASE_NOTES.md`, `MEMORY_BANK_OPTIMIZATIONS.md` и другие файлы документации

**Команды для копирования:**

```bash
# Windows PowerShell
# После клонирования репозитория в temp-memory-bank:
Copy-Item -Path "temp-memory-bank\.cursor" -Destination "ваш-проект\.cursor" -Recurse
Copy-Item -Path "temp-memory-bank\.cursorindexingignore" -Destination "ваш-проект\.cursorindexingignore"

# Linux/Mac
cp -r temp-memory-bank/.cursor ваш-проект/
cp temp-memory-bank/.cursorindexingignore ваш-проект/
```

### Шаг 3: Настройка Custom Modes в Cursor

1. Откройте настройки Cursor (Settings)
2. Найдите раздел "Custom Modes" или "Rules"
3. Добавьте следующие режимы:
   - **`/van`** — укажите путь к файлу правил для инициализации
   - **`/plan`** — укажите путь к файлу правил для планирования
   - **`/creative`** — укажите путь к файлу правил для творческого этапа
   - **`/build`** — укажите путь к файлу правил для реализации
   - **`/reflect`** — укажите путь к файлу правил для рефлексии
   - **`/archive`** — укажите путь к файлу правил для архивации

### Шаг 4: Создание структуры Memory Bank

В корне проекта создайте директорию `memory-bank/` и следующие файлы:

```
memory-bank/
├── projectbrief.md          # Основной документ проекта
├── productContext.md         # Контекст продукта
├── activeContext.md          # Активный контекст работы
├── systemPatterns.md         # Архитектурные паттерны
├── techContext.md            # Технологический контекст
├── progress.md               # Отслеживание прогресса
├── tasks.md                  # Список задач
├── memory_bank_instructions.md # Инструкции для AI
├── creative/                 # Директория для креативных материалов
│   └── (будут добавляться файлы в процессе работы)
└── archive/                  # Архив завершенных задач
    └── (будут добавляться файлы после завершения)
```

**Быстрое создание структуры:**

```bash
# Windows PowerShell
mkdir memory-bank
cd memory-bank
New-Item projectbrief.md, productContext.md, activeContext.md, systemPatterns.md, techContext.md, progress.md, tasks.md, memory_bank_instructions.md
mkdir creative, archive

# Linux/Mac
mkdir -p memory-bank/{creative,archive}
cd memory-bank
touch projectbrief.md productContext.md activeContext.md systemPatterns.md techContext.md progress.md tasks.md memory_bank_instructions.md
```

### Шаг 5: Инициализация проекта

1. Откройте Cursor в вашем проекте
2. Используйте команду `/van` для инициализации
3. Следуйте инструкциям AI для заполнения базовых файлов Memory Bank

### Шаг 6: Адаптация правил (опционально)

Если нужно адаптировать правила под ваш рабочий процесс:

1. Попросите Cursor AI помочь модифицировать правила
2. Укажите специфические требования вашего проекта
3. Сохраните измененные правила

## Проверка установки

### Тест 1: Проверка команд

Попробуйте использовать команды в Cursor:

- `/van` — должна инициализировать проект
- `/plan` — должна создать план работы
- `/build` — должна начать процесс разработки

### Тест 2: Проверка файлов

Убедитесь, что:

- Директория `memory-bank/` создана
- Все необходимые файлы существуют
- Файлы обновляются при использовании команд

### Тест 3: Проверка контекста

1. Используйте команду `/plan` для создания задач
2. Закройте и откройте Cursor снова
3. Используйте команду `/build`
4. Убедитесь, что AI помнит созданные задачи (контекст сохранился)

## Решение проблем

### Команды не работают

- Убедитесь, что Custom Modes настроены в Cursor
- Проверьте пути к файлам правил
- Убедитесь, что используется Cursor 2.0 или новее

### Файлы Memory Bank не обновляются

- Проверьте права доступа к директории `memory-bank/`
- Убедитесь, что AI имеет доступ к файлам
- Попробуйте явно указать AI обновить файлы

### Конфликты с существующими инструментами

- Адаптируйте правила Memory Bank под ваш workflow
- Используйте только нужные команды
- Можно создать алиасы для команд, если нужно

## Примечания

- Memory Bank System — это система правил, а не исполняемый код
- Не требуются зависимости или установка пакетов
- Правила можно и нужно адаптировать под конкретный проект
- Система работает через команды Cursor, а не через API или библиотеки
