# Callouts (Блоки примечаний) в Markdown

## Что такое Callouts?

Callouts — это специальные блоки в Markdown, которые выделяют важную информацию, предупреждения, советы и т.д. Они делают заметки более структурированными и наглядными.

Пример callout-блока:

```
!!!NOTE это !!!NOTE<br>строка 2
```

!!!NOTE это !!!NOTE<br>строка 2

### Основные типы callouts

- `note` — обычная заметка или примечание
- `info` — информационный блок
- `tip` — полезный совет или рекомендация
- `hint` — подсказка (аналог tip)
- `important` — важная информация
- `warning` — предупреждение или предостережение
- `caution` — предостережение (аналог warning)
- `danger` — опасность или критическое предупреждение
- `error` — сообщение об ошибке
- `success` — успешное действие или результат
- `question` — вопрос или запрос на обратную связь
- `example` — пример использования
- `quote` — цитата или выдержка
- `abstract` — краткое содержание
- `bug` — известная проблема или баг
- `fail` — информация о неудаче или ошибке
- `todo` — задача для выполнения

!!!TIP Каждый тип callout обычно имеет свой уникальный цвет и иконку для лучшей визуальной идентификации

!!!NOTE Поддержка конкретных типов callout может отличаться в зависимости от используемого расширения Markdown

## Использование Callouts в VSCode

### 1. Поддержка Callouts

VSCode по умолчанию не поддерживает callouts в Markdown-превью. Для их отображения используйте расширения:

- **Markdown Preview Enhanced**
- **Obsidian-style Callouts** (если доступно)

### 2. Установка расширения

1. Откройте VSCode.
2. Перейдите в Extensions (`Ctrl+Shift+X`).
3. Найдите и установите `Markdown Preview Enhanced`.

### 3. Настройка расширения

- Откройте командную палитру (`Ctrl+Shift+P`).
- Введите `Markdown Preview Enhanced: Open Preview to the Side`.
- В настройках расширения можно включить поддержку callouts:

  - Через глобальный settings.json

    - Откройте файл настроек VSCode (`Ctrl+Shift+P` → "Preferences: Open Settings (JSON)")

  - Через локальный конфиг

    - Откройте или создайте файл `.vscode/settings.json` в корне проекта

      !!!NOTE Локальные настройки в `.vscode/settings.json` будут применяться только к текущему проекту

  - Добавьте следующие настройки:

```json
"markdown-preview-enhanced": {
	"enableTypographer": true, // Включает типографские улучшения текста
	"previewTheme": "github-dark.css", // Устанавливает темную тему GitHub для предпросмотра
	"frontMatterRender": true, // Включает отображение YAML frontmatter
	"enableScriptExecution": true, // Разрешает выполнение скриптов в markdown
	"style": "github-dark.css" // Устанавливает стиль оформления как github-dark
}
```

**Подробнее о каждой настройке**

- `enableTypographer`: Когда включено, улучшает типографику текста (например, заменяет прямые кавычки на типографские, тире на длинное тире и т.д.)
- `previewTheme`: Определяет тему для предпросмотра Markdown. В данном случае используется темная тема GitHub
- `frontMatterRender`: Позволяет отображать и обрабатывать YAML frontmatter (метаданные в начале markdown файла)
- `enableScriptExecution`: Разрешает выполнение встроенных скриптов в markdown-файлах (например, для диаграмм или математических формул)
- `style`: Задает основной CSS-стиль для оформления документа, аналогично `previewTheme`

!!!WARNING Включение `enableScriptExecution` может представлять риск безопасности, если вы открываете markdown-файлы из непроверенных источников

```json
"markdown": {
	"preview": {
		"breaks": true, // Одиночные переносы строк отображаются как <br>
		"typographer": true // Включает типографские улучшения текста
	}
}
```

**Подробнее о каждой настройке**

- `preview.breaks: true`
  - Позволяет использовать одиночные переносы строк в Markdown
  - Каждый перенос строки в исходном тексте будет отображаться как перенос в предпросмотре
  - Без этой настройки нужно ставить два пробела в конце строки или использовать пустую строку для переноса
- `preview.typographer: true`
  - Включает автоматические типографские улучшения текста
  - Заменяет прямые кавычки (") на типографские («»)
  - Преобразует дефисы (--) в типографское тире (—)
  - Улучшает отображение других спецсимволов

!!!TIP Эти настройки особенно полезны при написании документации или статей, где важно типографское качество текста

### 4. Кастомизация

Вы можете изменить стили callouts через пользовательские CSS-файлы, если расширение это поддерживает.

## Итог

Callouts делают заметки более информативными и структурированными. Для их использования в VSCode потребуется расширение, например, `Markdown Preview Enhanced`.
