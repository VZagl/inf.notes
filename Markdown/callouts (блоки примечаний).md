# Callouts (Блоки примечаний) в Markdown

## Что такое Callouts?

Callouts — это специальные блоки в Markdown, которые выделяют важную информацию, предупреждения, советы и т.д. Они делают заметки более структурированными и наглядными.

## Синтаксис GitHub-style callouts

```markdown
> [!NOTE]
> Это примечание

> [!WARNING]
> Это предупреждение

> [!TIP]
> Это подсказка
```

### Поддерживаемые типы callouts

GitHub поддерживает следующие типы callouts:

- `NOTE` — обычная заметка или примечание
- `TIP` — полезный совет
- `IMPORTANT` — важная информация
- `WARNING` — предупреждение
- `CAUTION` — предостережение

> [!NOTE]
> Каждый тип callout имеет свой уникальный цвет и иконку для лучшей визуальной идентификации

## Настройка VS Code для работы с callouts

### 1. Установка расширений

1. Откройте VS Code
2. Перейдите в Extensions (Ctrl+Shift+X)
3. Установите два расширения:
   - **Markdown All in One**
   - **Markdown Preview Github Styling**

### 2. Настройка

Добавьте следующие настройки в ваш `.vscode/settings.json`:

```json
{
	
	"markdown": {
		"preview": {
			"breaks": true, // Одиночные переносы строк отображаются как <br>
			"typographer": true // Включает типографские улучшения текста
		},
		"extension.preview.autoShowPreviewToSide": true // Автоматически открывать предпросмотр Markdown сбоку
	},

	"[markdown]": {
		"editor.defaultFormatter": "yzhang.markdown-all-in-one", // Использует Prettier для форматирования Markdown файлов
		"editor.formatOnSave": true, // Автоматическое форматирование при сохранении
		"editor.wordWrap": "on", // Включает перенос строк для удобства чтения
		"editor.renderWhitespace": "all", // Показывает символы пробелов и табуляции
		"editor.quickSuggestions": {
			// Настройки автодополнения
			"comments": "on", // Включено в комментариях
			"strings": "on", // Включено в строках
			"other": "on" // Включено везде
		}
	}
}
```

### 3. Использование

1. Откройте markdown файл
2. Нажмите Ctrl+K V для открытия предпросмотра
3. Callouts будут отображаться в GitHub-стиле

## Примеры использования

> [!NOTE]
> Важное примечание или дополнительная информация

> [!TIP]
> Полезный совет по использованию

> [!IMPORTANT]
> Критически важная информация

> [!WARNING]
> Предупреждение о возможных проблемах

> [!CAUTION]
> Предостережение о потенциально опасных действиях

## Ограничения

> [!NOTE]
> GitHub-style callouts поддерживают только 5 типов блоков, в отличие от других систем разметки, которые могут поддерживать больше вариантов

## Итог

Callouts в GitHub-стиле - это простой и эффективный способ выделить важную информацию в ваших markdown документах. Хотя количество поддерживаемых типов ограничено, их достаточно для большинства случаев использования.
