# [Промпты](https://docs.continue.dev/customize/deep-dives/prompts.md)

Промпты — это многократно используемые инструкции, к которым можно обращаться в любое время во время чата. Они особенно полезны в качестве контекста для повторяющихся и/или сложных задач.
<Info>
Ранее промпты определялись в файловом формате `.prompt`, но для
согласованности мы теперь рекомендуем использовать тот же формат Markdown, что и
[правила](./rules.mdx), и добавлять `alwaysApply: false` в фронтматтер, чтобы
они запускались вручную.
</Info>

### Быстрый старт

Ниже приведен быстрый пример настройки файла промпта:

1. Создайте папку с именем `.continue/rules` на верхнем уровне вашего рабочего пространства.
2. Добавьте файл с именем `review-prompt.md` в эту папку.
3. Напишите следующее содержимое в `review-prompt.md` и сохраните.

```md title="review-prompt.md"
---
name: Проверка лучших практик Redux
alwaysApply: false
---

Просмотрите текущий открытый файл на предмет соблюдения лучших практик Redux, как объясняется в их руководстве по стилю на https://redux.js.org/style-guide/.
```

Теперь, чтобы использовать этот промпт, вы можете открыть чат, нажать <kbd>/</kbd>, выбрать промпт и набрать любые дополнительные инструкции, которые вы хотели бы добавить.

## Дополнительные примеры

Ниже приведены дополнительные примеры для начала работы. Вы также можете посетить Hub, чтобы [изучить промпты](https://hub.continue.dev/explore/prompts) или [создать свои собственные](https://hub.continue.dev/new?type=block&blockType=prompts).

### Проверка безопасности

```md title="security-review.md"
---
name: Проверка лучших практик безопасности
alwaysApply: false
---

Проверьте изменения в текущем git diff на соответствие следующим лучшим практикам безопасности:

- Следует ли архитектура принципам безопасности по замыслу (security-by-design)?
- Есть ли потенциальные уязвимости безопасности в дизайне системы?
- Корректно ли обрабатываются конфиденциальные данные на протяжении всего жизненного цикла?
```
